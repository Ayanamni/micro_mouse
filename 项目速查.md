# 项目速查

本文件按需阅读，不属于每轮必读上下文。需要项目地图、启动命令、主动控制路径或历史坑点时再打开。

## 当前目标

MuJoCo 物理仿真用于高速巡线电子鼠控制器验证。当前阶段聚焦 v/omega 解耦控制器和真实光电管巡线；规划/感知后置。

## 主动控制路径

- Python 主入口：`scripts/workbench.py`
- 主控制 API：`control_core.vw_omega_step` / `control_core.vw_control_tick`
- 不要把 legacy `control_core.step` 当作主动路径。
- 控制结构：`VWController`，omega loop 5kHz，velocity/tracking 1kHz，5kHz duty refresh。
- 力矩混合：`tau_L = tau_v - tau_omega`，`tau_R = tau_v + tau_omega`。
- 默认参数源：`micromouse_sim/config/vw_controller_config.py`

## 常用命令

```bash
python scripts/workbench.py
python scripts/workbench.py --viewer-only --duration 15
python scripts/workbench.py --no-render --speed 2.0 --duration 10
python scripts/eval_vw.py omega-step --track 2019kansai --speed 1.0 --omega-step-value 5.0
```

用户暂停巡线前不要主动跑：

```bash
python scripts/eval_vw.py single --track 2019kansai --speed 3.5 --duration 12
python scripts/eval_vw.py sweep --track 2019kansai --speed-min 1.0 --speed-max 5.0
```

## 必记坑点

- `qfrc_applied` 使用 `=`，不是 `+=`；MuJoCo 会跨步保留值。
- `xfrc_applied[:]` 每步覆盖，不要在旧值上叠加外力。
- 左右轮 Z offset 必须一致，否则车会自旋。
- 最终 `u_L/u_R` 才 clamp 到 `[-1, 1]`；不要把 steer 错误限制在 throttle 内。
- 电机噪声当前默认关闭，真实性测试时再启用。

## 验证选择

- 只改文档：检查 diff 即可。
- 改 Python：至少 `py_compile` 相关文件。
- 改 C++/binding：必须编译。
- 改主动控制路径：跑对应 bench/eval。
- 调 omega loop：优先看 `omega-step` 的 rise、settling、bandwidth，再看 overshoot。
